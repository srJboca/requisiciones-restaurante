# General Rules
1. Always must separate the backend codebase from the frontend codebase.
2. The backend must run on a different port than the frontend.
3. The backend must have a Swagger documentation.
4. The databases must reside on a dedicated docker container.
5. The front end must run on a different docker container.
6. The app must have user authentication.
7. The app must have user roles.
8. The app must have permissions.
9. The app must have logging.
10. The app must have a audit trail.
11. The database must be MySQL.
12. The database must be relational.
13. The database must be accessible just for the backend (docker container).
14. The frontend must be accessible for any user
15. Must have a welcome page before user login.
